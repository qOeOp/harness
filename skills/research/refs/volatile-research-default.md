# Volatile Research Default

Canonical source:

- [docs/workflows/volatile-research-default.md](../../../docs/workflows/volatile-research-default.md)

Research-bundle checklist:

1. Treat external facts about current tools, APIs, pricing, regulations, community best practice, and recent repository activity as `volatile-by-default`.
2. Do not turn those claims into formal requirements or recommendations without fresh evidence.
3. When freshness matters, create or reference a `research dispatch` first.
4. Prefer official docs, official APIs, or source notes over broader web material.
5. If freshness cannot be established, mark the result as `exploratory` or `blocked by freshness`.
