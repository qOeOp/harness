---
name: research-memo
description: Use when a vague problem needs to be turned into a structured research memo with evidence, counter-evidence, and unknowns.
---

Create a research memo that separates:

- evidence
- counter-evidence
- unknowns
- recommendation

Use [../../docs/templates/research-memo.md](../../docs/templates/research-memo.md).
Read [../../docs/workflows/volatile-research-default.md](../../docs/workflows/volatile-research-default.md).
Read [../../docs/workflows/internal-research-routing.md](../../docs/workflows/internal-research-routing.md).
Read [../../docs/workflows/task-artifact-routing.md](../../docs/workflows/task-artifact-routing.md).
For volatile topics, create or reference a research dispatch, then gather fresh external sources or source-note references before recommending.
If a work item is in play, write to `.harness/tasks/<task-id>/refs/`.
Only when `advanced governance mode` is explicitly enabled and the memo is truly cross-task, promote it to a governance workspace location.
