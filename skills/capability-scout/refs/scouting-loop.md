# Scouting Loop

Treat capability scouting as a repeatable pipeline, not as ad hoc browsing.

## Loop

1. define target outcome
2. define hard constraints
3. scan likely sources
4. shortlist candidates
5. inspect architecture, tooling, artifact model, and maintenance signal
6. choose reuse mode
7. write absorption plan

## Hard rule

Do not confuse `interesting` with `absorbable`.

A strong candidate is one whose useful slice can be expressed locally with clear ownership, bounded files, and a maintainable runtime contract.
